## 2. What was built

| Component | File | Description |
|---|---|---|
| Proto contract | `proto/kb.proto` | 6 RPCs (`GetProcessState`, `ListZone`, `SetContainment`, `StreamEvents`, `SubmitAgentDecision`, `StreamAlerts`), shared `Zone`/`ContainmentLevel` enums |
| Codegen pipeline | `Makefile` (`make proto`) | Drives `protoc` + `protoc-gen-go` + `protoc-gen-go-grpc` |
| Daemon entrypoint | `cmd/kbd/main.go` | Cobra CLI, `--config` flag, graceful SIGINT/SIGTERM shutdown |
| Control plane core | `internal/controlplane/controlplane.go` | In-memory process state store, EMA-smoothed scoring, zone classification, all 6 gRPC handlers implemented, event/alert fan-out to live stream subscribers |
| Config loader | `internal/controlplane/config.go` | Reads `config/kb.yaml`, falls back to defaults if absent |
| Config files | `config/kb.yaml`, `config/policy.yaml`, `config/allowlist.yaml` | Match the schema documented in `config/README.md` |
| Unit tests | `internal/controlplane/controlplane_test.go` | EMA math, zone-boundary classification, sustained-score zone transition — pure logic, no network/DB dependency |

**Design notes beyond the original draft:**
- Zone thresholds (40 / 75) and EMA `alpha` (0.3) are config-driven, not hardcoded.
- `StreamEvents`/`StreamAlerts` actually fan out to subscribers now (the original draft logged events but never delivered them to stream listeners).
- `SubmitAgentDecision` enforces a first-pass version of the design doc's **AI Dependency Constraint** (Sec 7.1): a decision with no `authorized_by` is rejected rather than silently accepted.

## 3. Build issues hit and resolved

| Issue | Cause | Fix |
|---|---|---|
| `protoc-gen-go: program not found` | Plugin not on `PATH` | `go install` both plugins, added `$(go env GOPATH)/bin` to `PATH` |
| `go mod tidy` tried to fetch `.../kb-control-plane/proto` from GitHub | `Makefile`'s `--go_out=proto` combined with `paths=source_relative` doubled the path, generating into `proto/proto/` and leaving the real `proto/` empty of `.go` files | Changed `--go_out`/`--go-grpc_out` to `.` (repo root) |

## 4. Explicitly not done yet (by design, not oversight)

These are marked as `TODO` inline in the code and tracked in the README status table:

- **Persistent state** — store is in-memory only; restarts lose all process history. SQLite integration (`mattn/go-sqlite3` is already in `go.mod`) is next.
- **`internal/enforcement`** — `applyContainment`/`applyMonitoring`/`relaxContainment` are log-only stubs; no real namespace/seccomp/cgroup primitives yet.
- **`internal/audit`** — no SHA-256 tamper-evident chain yet; `SetContainment` and `SubmitAgentDecision` both have TODOs flagging where audit entries belong.
- **`internal/scoring`** — the six weighted behavioral dimensions (syscall entropy, privilege delta, lineage, memory, network, file access) aren't computed anywhere yet. `UpdateScore()` and the EMA math are built and unit-tested, but nothing in the event pipeline calls it — `handleEvent` currently just logs and broadcasts raw events.
- **Policy engine** — `config/policy.yaml` and `config/allowlist.yaml` exist as documented schema but aren't read/evaluated by any code yet.

## 5. Suggested next milestone

Natural next slice, in rough dependency order:
1. `internal/scoring` — implement the weighted composite + wire it into `handleEvent` so `UpdateScore` actually gets called from real events.
2. `internal/audit` — SHA-256 hash-chained log; lowest-risk to build in isolation and easy to unit test (no kernel/eBPF dependency).
3. `internal/enforcement` — first real primitive (seccomp net-block for Suspicious zone is the smallest one).
4. SQLite state store to replace the in-memory map.
